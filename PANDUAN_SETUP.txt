## 2ï¸âƒ£ CARA SETUP SUPABASE (10-15 menit)

### LANGKAH 1: Buat Akun Supabase

1. Buka browser, pergi ke: https://supabase.com
2. Klik tombol "Start your project" atau "Sign Up"
3. Daftar menggunakan akun GitHub (paling mudah)
   - Jika belum punya GitHub, daftar dulu di https://github.com

### LANGKAH 2: Buat Project Baru

1. Setelah login, klik "New Project"
2. Isi formulir:
   - Name: digimart (atau nama apapun)
   - Database Password: buat password yang kuat (SIMPAN BAIK-BAIK!)
   - Region: pilih yang terdekat (Southeast Asia - Singapore)
3. Klik "Create new project"
4. Tunggu 1-2 menit sampai project siap

### LANGKAH 3: Jalankan Database Schema

1. Di dashboard Supabase, klik menu "SQL Editor" di sidebar kiri
2. Klik "New Query"
3. Buka file: d:\webUAS\supabase\schema.sql
4. Copy SEMUA isi file tersebut
5. Paste ke SQL Editor di Supabase
6. Klik tombol "Run" atau tekan Ctrl+Enter
7. Jika berhasil, akan muncul pesan "Success"

### LANGKAH 4: Buat Storage Bucket

1. Klik menu "Storage" di sidebar kiri
2. Klik "Create a new bucket"
3. Isi:
   - Name: digital-assets
   - PENTING: JANGAN centang "Public bucket" (biarkan private!)
4. Klik "Create bucket"

### LANGKAH 5: Ambil API Keys

1. Klik ikon âš™ï¸ "Settings" di sidebar kiri bawah
2. Klik "API" di submenu
3. Kamu akan melihat:

## 3ï¸âƒ£ CARA MENGISI ENVIRONMENT VARIABLES
â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

Environment Variables adalah "pengaturan rahasia" yang 
TIDAK boleh diupload ke GitHub (karena berisi API keys).

### LANGKAH 1: Buat File .env.local

1. Buka folder: d:\webUAS\digimart
2. Buat file baru dengan nama: .env.local
   (Ingat, ada titik di depan nama file!)

### LANGKAH 2: Isi dengan API Keys

Copy-paste ini ke file .env.local, lalu ganti nilai-nilainya:

```
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGc...xxxxx
SUPABASE_SERVICE_ROLE_KEY=eyJhbGc...xxxxx

# Website URL
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

Ganti "xxxxx" dengan nilai asli dari Supabase dashboard.

### LANGKAH 3: Simpan File

Tekan Ctrl+S untuk menyimpan.

âš ï¸ PENTING: File .env.local sudah otomatis di-ignore oleh Git,
jadi tidak akan terupload ke GitHub. JANGAN ubah ini!

## 4ï¸âƒ£ CARA SETUP PAYMENT GATEWAY (OPSIONAL)
â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

Untuk menerima pembayaran QRIS di Indonesia, kamu bisa 
menggunakan salah satu layanan ini:

### OPSI A: MIDTRANS (Paling Populer di Indonesia)
Website: https://midtrans.com

1. Daftar akun Midtrans
2. Verifikasi bisnis (butuh dokumen usaha)
3. Dapatkan Server Key dan Client Key
4. Midtrans akan generate QR Code QRIS otomatis

### OPSI B: XENDIT
Website: https://www.xendit.co/id/

1. Daftar akun Xendit
2. Verifikasi identitas
3. Aktifkan metode pembayaran QRIS
4. Dapatkan API Key

### OPSI C: MANUAL (Untuk Testing)
Untuk testing, kamu bisa:
1. Gunakan QR Code dari DANA/OVO/GoPay merchant
2. Verifikasi pembayaran secara manual

### INTEGRASI (Contoh dengan Midtrans)

Setelah punya akun Midtrans, tambahkan ke .env.local:

```
MIDTRANS_SERVER_KEY=SB-Mid-server-xxxxx
MIDTRANS_CLIENT_KEY=SB-Mid-client-xxxxx
```

Q: Apakah Supabase gratis?
A: Ya! Untuk project kecil-menengah, tier gratis sudah cukup.
   Limit: 500MB database, 1GB storage, 50K monthly users.

Q: Bagaimana kalau website sudah jadi dan mau online?
A: Upload ke Vercel (vercel.com) - gratis dan mudah!
   1. Push code ke GitHub
   2. Connect GitHub dengan Vercel
   3. Vercel akan auto-deploy

Q: Bagaimana cara menambah produk baru?
A: Buka Supabase Dashboard > Table Editor > products
   Klik "Insert row" dan isi data produk.

Q: File digital disimpan dimana?
A: Di Supabase Storage bucket "digital-assets"
   Upload file melalui Storage > digital-assets > Upload
